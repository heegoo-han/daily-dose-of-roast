## Overview
Trello 스타일의 칸반 Todo 앱. 3개 고정 칼럼에서 카드를 드래그&드롭으로 이동하고, 검색·필터로 원하는 카드를 빠르게 찾으며, localStorage로 데이터를 영속화한다.

## 시나리오

### 1. 카드 생성
[상황] 사용자가 칼럼의 "추가" 버튼을 클릭하고 제목을 입력하면
[동작] 해당 칼럼 하단에 새 카드가 추가된다

성공 기준:
- [ ] 제목 "회의록 작성" 입력 → "To Do" 칼럼에 "회의록 작성" 카드가 표시됨
- [ ] 제목을 비운 채 추가 시도 → "제목을 입력해주세요" 에러 메시지 표시됨
- [ ] 새 카드의 기본 우선순위는 "중"으로 설정됨

### 2. 카드 인라인 편집
[상황] 사용자가 카드를 클릭하면
[동작] 카드의 모든 필드(제목, 설명, 우선순위, 태그, 마감일)를 인라인으로 편집할 수 있다

성공 기준:
- [ ] 카드 클릭 → 제목, 설명, 우선순위, 태그, 마감일 편집 UI가 표시됨
- [ ] 제목을 "회의록 작성"에서 "주간 회의록"으로 변경 → 카드에 "주간 회의록"이 표시됨
- [ ] 우선순위를 "중"에서 "상"으로 변경 → 카드에 "상" 우선순위가 표시됨
- [ ] 제목을 빈 값으로 변경 시도 → "제목을 입력해주세요" 에러 메시지 표시됨

### 3. 카드 삭제
[상황] 사용자가 카드의 삭제 버튼을 클릭하면
[동작] 확인 대화상자가 표시되고, 확인 시 카드가 삭제된다

성공 기준:
- [ ] 삭제 버튼 클릭 → "정말 삭제하시겠습니까?" 확인 대화상자 표시됨
- [ ] 확인 클릭 → 카드가 목록에서 사라짐
- [ ] 취소 클릭 → 카드가 그대로 유지됨

### 4. 칼럼 간 드래그&드롭
[상황] 사용자가 카드를 다른 칼럼으로 드래그하면
[동작] 카드가 해당 칼럼으로 이동한다

성공 기준:
- [ ] "To Do"의 카드를 "In Progress"로 드래그 → "In Progress" 칼럼에 카드가 표시됨
- [ ] 같은 칼럼 내에서 카드를 드래그 → 카드 순서가 변경됨

### 5. 태그 관리
[상황] 사용자가 카드 편집 시 태그를 입력하면
[동작] 새 태그가 생성되거나 기존 태그 목록에서 선택된다

성공 기준:
- [ ] "버그" 태그 입력 → 카드에 "버그" 태그가 표시됨
- [ ] 기존에 "버그" 태그가 있을 때 "버"를 입력 → 자동완성 목록에 "버그"가 표시됨
- [ ] 카드에서 "버그" 태그 제거 → 카드에서 태그가 사라짐

### 6. 제목 검색
[상황] 사용자가 검색창에 텍스트를 입력하면
[동작] 제목에 해당 텍스트가 포함된 카드만 표시된다

성공 기준:
- [ ] "회의"를 검색 → "회의록 작성" 카드만 표시되고 나머지는 숨겨짐
- [ ] 검색어를 지우면 → 모든 카드가 다시 표시됨

### 7. 우선순위·태그 필터
[상황] 사용자가 필터 옵션에서 우선순위 또는 태그를 선택하면
[동작] 해당 조건에 맞는 카드만 표시된다

성공 기준:
- [ ] 우선순위 "상" 필터 선택 → 우선순위가 "상"인 카드만 표시됨
- [ ] 태그 "버그" 필터 선택 → "버그" 태그가 있는 카드만 표시됨
- [ ] 우선순위 "상" + 태그 "버그" 동시 선택 → 두 조건을 모두 만족하는 카드만 표시됨
- [ ] 필터 해제 → 모든 카드가 다시 표시됨

### 8. 다크모드 토글
[상황] 사용자가 다크모드 토글 버튼을 클릭하면
[동작] 앱 전체 테마가 라이트/다크로 전환된다

성공 기준:
- [ ] 라이트모드에서 토글 클릭 → 다크모드로 전환됨
- [ ] 다크모드에서 토글 클릭 → 라이트모드로 전환됨
- [ ] 테마 설정이 새로고침 후에도 유지됨

### 9. 데이터 영속화 (localStorage)
[상황] 사용자가 카드를 추가/수정/삭제/이동하면
[동작] 변경 사항이 localStorage에 자동 저장되고, 페이지 새로고침 시 복원된다

성공 기준:
- [ ] 카드 추가 후 새로고침 → 추가한 카드가 그대로 표시됨
- [ ] 카드를 다른 칼럼으로 이동 후 새로고침 → 이동한 칼럼에 카드가 표시됨

### 10. JSON 내보내기
[상황] 사용자가 내보내기 버튼을 클릭하면
[동작] 현재 보드의 모든 데이터가 JSON 파일로 다운로드된다

성공 기준:
- [ ] 내보내기 클릭 → JSON 파일이 다운로드됨
- [ ] 다운로드된 파일에 모든 칼럼과 카드 데이터가 포함됨

### 11. JSON 가져오기
[상황] 사용자가 가져오기 버튼을 클릭하고 JSON 파일을 선택하면
[동작] 확인 대화상자가 표시되고, 확인 시 기존 데이터가 교체된다

성공 기준:
- [ ] 파일 선택 후 "기존 데이터가 모두 교체됩니다" 확인 대화상자 표시됨
- [ ] 확인 클릭 → 가져온 데이터로 보드가 교체됨
- [ ] 취소 클릭 → 기존 데이터가 유지됨
- [ ] 잘못된 형식의 파일 가져오기 → "올바른 형식의 파일이 아닙니다" 에러 메시지 표시됨

## 범위

### 포함
- 3개 고정 칼럼 (To Do / In Progress / Done) 칸반 보드
- 카드 CRUD (제목, 설명, 우선순위, 태그, 마감일)
- 카드 인라인 편집
- 칼럼 간/칼럼 내 드래그&드롭
- 사용자 생성 태그 (자동완성 포함)
- 제목 검색 (실시간 필터링)
- 우선순위·태그 필터 (AND 조합)
- 다크모드 토글 (설정 영속화)
- localStorage 자동 저장·복원
- JSON 내보내기/가져오기 (전체 교체 방식)

### 제외
- 사용자 인증/로그인: 단일 사용자 로컬 앱이므로 불필요
- 서버 동기화/실시간 협업: localStorage 기반이므로 서버 불필요
- 칼럼 추가/삭제/이름 변경: 3개 고정 칼럼으로 범위 한정
- 카드 내 첨부파일/이미지: 텍스트 기반 카드로 범위 한정
- 알림/리마인더: MVP 범위 외
