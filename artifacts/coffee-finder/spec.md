## Overview
현재 위치 기반으로 주변 커피 가게를 Neo-Brutalism 스타일로 화려하게 탐색하는 앱. 목업 데이터로 스크린샷·대표메뉴·평점·거리·추천이유를 한 화면에서 확인한다.

## 시나리오

### 1. 커피 가게 목록 초기 표시
[상황] 사용자가 앱에 진입하면
[동작] 시스템이 목업 커피 가게 목록을 평점 높은 순으로 카드 형태로 보여준다

성공 기준:
- [ ] 앱 진입 → 커피 가게 카드가 1개 이상 표시된다
- [ ] 첫 번째 카드의 평점이 두 번째 카드의 평점보다 높거나 같다
- [ ] 각 카드에 가게 사진(이미지), 대표 메뉴, 평점, 거리, 추천 이유가 모두 표시된다

### 2. 평점 순 정렬
[상황] 정렬 옵션이 표시된 커피 가게 목록에서
[동작] 사용자가 "평점 순" 정렬을 선택하면
[동작] 시스템이 평점 높은 순으로 카드를 재배열한다

성공 기준:
- [ ] "평점 순" 선택 → 평점 4.8인 카드가 평점 4.3인 카드보다 위에 표시된다
- [ ] 현재 선택된 정렬 기준이 활성 상태로 강조 표시된다

### 3. 거리 순 정렬
[상황] 정렬 옵션이 표시된 커피 가게 목록에서
[동작] 사용자가 "거리 순" 정렬을 선택하면
[동작] 시스템이 가까운 순으로 카드를 재배열한다

성공 기준:
- [ ] "거리 순" 선택 → 150m 카드가 800m 카드보다 위에 표시된다
- [ ] 현재 선택된 정렬 기준이 활성 상태로 강조 표시된다

### 4. 인기 순 정렬
[상황] 정렬 옵션이 표시된 커피 가게 목록에서
[동작] 사용자가 "인기 순" 정렬을 선택하면
[동작] 시스템이 리뷰 수 많은 순으로 카드를 재배열한다

성공 기준:
- [ ] "인기 순" 선택 → 리뷰 1200개 카드가 리뷰 300개 카드보다 위에 표시된다
- [ ] 현재 선택된 정렬 기준이 활성 상태로 강조 표시된다

### 5. 카드에 모든 핵심 정보 표시
[상황] 커피 가게 목록이 표시된 상태에서
[동작] 사용자가 카드를 바라보면
[동작] 시스템이 가게 사진, 대표 메뉴, 평점, 거리, 추천 이유를 한 카드 안에 보여준다

성공 기준:
- [ ] 카드에 가게 이미지가 표시된다
- [ ] 카드에 대표 메뉴 이름(예: "아이스 아메리카노")이 표시된다
- [ ] 카드에 평점 숫자(예: "4.7")가 표시된다
- [ ] 카드에 거리 텍스트(예: "350m")가 표시된다
- [ ] 카드에 추천 이유 텍스트(예: "원두 직접 로스팅, 산미 없는 진한 맛")가 표시된다

## 범위

### 포함
- 목업 데이터 기반 커피 가게 목록 (최소 5개)
- 평점 순 / 거리 순 / 인기 순 정렬 전환
- 카드당 표시: 가게 사진, 대표 메뉴, 평점, 거리, 추천 이유
- Neo-Brutalism 스타일 (굵은 테두리, 강한 그림자, 고대비 색상)

### 제외
- 실제 GPS 위치 API 연동 — 목업 데이터로 충분한 UI 검증이 목적이므로
- 카드 클릭 상세 페이지 — 목록 카드에 모든 정보를 담기로 결정
- 검색/필터 기능 — 1차 범위 초과, 정렬만으로 탐색 가능
- 즐겨찾기/저장 기능 — 1차 범위 초과

## 전제 조건
- 목업 커피 가게 데이터에 name, image, menuName, rating, reviewCount, distance, recommendation 필드가 있다
- 거리 단위는 미터(m) 또는 킬로미터(km)로 표시한다
- 평점은 소수점 1자리(예: 4.7)로 표시한다
